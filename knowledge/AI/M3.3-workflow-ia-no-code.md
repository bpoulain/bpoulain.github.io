# M3.3 — Construire un workflow IA sans coder : outils no-code/low-code

> En 2025, il est possible de connecter un LLM à ses outils existants, de construire des automatisations IA, et de prototyper des agents — sans écrire une seule ligne de code. Ce module donne les bases pratiques pour le faire.

---

## 1. Théorie

### Définition

Un workflow IA no-code est une automatisation construite avec des outils visuels (blocs, connexions, conditions) qui intègre un ou plusieurs modèles IA (LLM, classification, vision) dans une séquence d'actions : lire un email → résumer avec un LLM → créer un ticket → notifier sur Slack. Ces workflows peuvent remplacer des heures de traitement manuel répétitif.

### Modèles mentaux

**Modèle 1 : Le workflow comme chaîne de responsabilités**
Un workflow est une séquence de blocs, chaque bloc étant responsable d'une seule action :
- **Trigger** : ce qui déclenche le workflow (email reçu, formulaire soumis, heure programmée)
- **Action** : ce que fait chaque étape (lire, transformer, appeler une IA, envoyer, créer)
- **Condition** : une bifurcation selon un résultat (si l'IA classe "urgent" → notifier ; sinon → traiter normalement)
- **Sortie** : le résultat final du workflow (document créé, ticket ouvert, email envoyé)

**Modèle 2 : Les 3 types de blocs IA dans un workflow**
1. **Génération** : le LLM produit du contenu à partir d'un input (résumé, rédaction, traduction)
2. **Classification** : le LLM classe un input dans une catégorie (urgent/normal, positif/négatif, type A/B/C)
3. **Extraction** : le LLM extrait des informations structurées depuis du texte non structuré (nom, montant, date depuis un email)

### Schéma conceptuel

```
EXEMPLE DE WORKFLOW : Triage de demandes support

TRIGGER
  ↓ Email reçu dans support@monproduit.com
  
ÉTAPE 1 — Extraction IA
  ↓ LLM extrait : sujet, urgence déclarée, type de problème
  
ÉTAPE 2 — Classification IA
  ↓ LLM classe : Bug critique / Question usage / Facturation / Autre
  
ÉTAPE 3 — Condition
  ├── Si "Bug critique" → 
  │     Créer ticket Jira HIGH PRIORITY
  │     Notifier #incidents sur Slack
  │     Envoyer accusé de réception personnalisé
  │
  ├── Si "Question usage" →
  │     LLM génère réponse depuis la documentation
  │     Envoyer réponse automatique
  │     Archiver dans Notion
  │
  └── Si autre →
        Créer ticket standard
        Assigner au prochain agent disponible

SORTIE
  Ticket créé + notification équipe + client informé
  Durée : automatique en < 30 secondes
```

### Analogie

"Construire un workflow no-code, c'est comme assembler des Lego : chaque brique fait une chose précise, et on les connecte dans l'ordre voulu. Le LLM est une brique spéciale qui peut lire, comprendre et transformer du texte — on la place où on a besoin d'intelligence dans la chaîne."

## 2. Compréhension

### 5 questions de vérification

1. Quelle est la différence entre un trigger "à la demande" et un trigger "automatique" dans un workflow ?
2. Quand préférer un bloc de classification IA plutôt qu'une condition if/else classique ?
3. Pourquoi les workflows IA doivent-ils inclure une gestion des erreurs explicite ?
4. Comment tester un workflow IA avant de le mettre en production ?
5. Quelle est la limite principale des outils no-code par rapport à du code custom ?

### Pièges courants

- **Workflows trop complexes** : un workflow de 20 étapes sans documentation est impossible à maintenir. Décomposer en sous-workflows.
- **Ignorer la gestion des erreurs** : que se passe-t-il si le LLM ne répond pas ? Si l'API est down ? Le workflow doit gérer ces cas.
- **Coût des tokens LLM** : chaque appel à un LLM coûte des tokens. Un workflow qui tourne 1000 fois/jour avec un gros prompt peut coûter cher.
- **Données sensibles** : attention aux informations personnelles envoyées à des APIs LLM externes (RGPD, sécurité).
- **Drift de qualité** : un workflow qui fonctionnait bien peut se dégrader si les inputs changent de format ou si le modèle est mis à jour.

### Limites du concept

Les outils no-code ont des limites : personnalisation réduite, coût par action élevé à grand volume, moins de contrôle sur les erreurs. Pour des workflows critiques ou à très grande échelle, le code reste souvent préférable.

## 3. Application Produit / Design

### Cas d'usage concrets

**Cas 1 : Synthèse hebdomadaire des retours utilisateurs**
Trigger : chaque lundi à 8h / Sources : Typeform, Intercom, App Store reviews / Étapes : collecte des verbatims de la semaine → LLM synthèse les thèmes principaux → LLM génère un rapport formaté → Envoi dans #product-insights Slack + création d'une page Notion.
Outil : Make ou n8n. Durée de setup : 2-3h.

**Cas 2 : Qualification automatique de leads entrants**
Trigger : soumission formulaire HubSpot / Étapes : LLM analyse le formulaire → extrait secteur, taille, besoin → classifie la priorité (A/B/C) → si A : notifier le commercial en Slack + créer tâche urgente / si B/C : enroller dans séquence email automatique.
Outil : Make + GPT-4. Durée de setup : 3-4h.

**Cas 3 : Transcription et synthèse automatique de réunions**
Trigger : fin de réunion Zoom / Étapes : récupérer la transcription → LLM extrait les décisions, les actions, les questions ouvertes → formater selon template → créer une page Notion + envoyer le résumé aux participants.
Outil : Zapier + Claude API. Durée de setup : 1-2h.

### Traduction IA → processus

| Processus manuel | Workflow IA no-code équivalent |
|---|---|
| Lire + trier les emails support | Trigger email → Classification LLM → Routing automatique |
| Résumer les entretiens utilisateurs | Trigger Otter/transcription → Résumé LLM → Export Notion |
| Rédiger les release notes | Trigger merge PR → Extraction LLM → Draft release notes |
| Analyser les retours App Store | Trigger schedulé → Scraping → Synthèse LLM → Rapport Slack |

## 4. Pratique

### Exercice 1 — Cartographier avant de builder (20 min)

Avant d'ouvrir un outil, documente sur papier le workflow que tu veux construire :
1. Trigger : qu'est-ce qui déclenche ce workflow ?
2. Étapes : liste chaque action dans l'ordre
3. Données qui transitent : qu'est-ce qui passe d'une étape à l'autre ?
4. Branchements : y a-t-il des conditions ?
5. Sorties : que produit le workflow ?
6. Erreurs : que se passe-t-il si une étape échoue ?

### Exercice 2 — Construire ton premier workflow IA (30 min)

Construis le workflow suivant dans Make (compte gratuit) :
1. **Trigger** : email reçu dans Gmail avec le sujet "Feedback produit"
2. **Étape 1** : envoyer le corps de l'email à Claude/GPT
   - Prompt : "Résume ce feedback en 3 points. Identifie si c'est positif, négatif, ou neutre."
3. **Étape 2** : créer une ligne dans un Google Sheet avec : date, expéditeur, résumé, sentiment
4. **Bonus** : si sentiment = "négatif", envoyer une notification Slack

*Critère de réussite : le workflow tourne sans erreur sur 3 emails de test.*

### Mini-projet

**"Workflow IA pour mon équipe"**

Construis un workflow IA qui automatise une vraie tâche de ton équipe (identifiée en M3.1). Documentation attendue :
1. Schéma du workflow (blocs + connexions)
2. Screenshot de l'outil no-code utilisé
3. Exemple d'output pour 1 run
4. Temps économisé estimé par semaine
5. Limites et points d'amélioration

*Livrable : workflow fonctionnel documenté, prêt à être partagé avec l'équipe.*

## 6. Implémentation technique

### Outils recommandés

| Outil | Points forts | Gratuit | Niveau |
|---|---|---|---|
| Make (ex-Integromat) | Interface visuelle, nombreux connecteurs, LLM intégré | Oui (limité) | Débutant |
| Zapier | Le plus simple, énorme bibliothèque d'apps | Oui (très limité) | Débutant |
| n8n | Open source, self-hostable, très flexible | Oui (self-host) | Intermédiaire |
| Notion AI | Workflows légers dans Notion | Avec abonnement | Débutant |
| Dify.ai | Spécialisé workflows LLM, RAG inclus | Oui (limité) | Intermédiaire |
| Relevance AI | Focus agents et workflows IA complexes | Oui (limité) | Intermédiaire |

### APIs LLM à connecter

- **Anthropic API (Claude)** : meilleur pour les tâches longues et complexes
- **OpenAI API (GPT-4o)** : polyvalent, grand écosystème
- **Groq (LLaMA)** : très rapide, gratuit en développement
