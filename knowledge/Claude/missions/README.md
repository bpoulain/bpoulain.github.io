# Missions pratiques — Scénarios guidés

4 missions complètes avec une entreprise fictive pour pratiquer Claude Code en conditions réelles.

## L'entreprise fictive : **Pixio**

**Pixio** est une startup SaaS B2B en phase de croissance (Series A, 18 mois d'existence).

**Produit** : Plateforme de création et distribution de micro-contenus visuels pour les équipes marketing (social media, newsletters, présentations).

**Chiffres clés** :
- 1 200 clients payants (plan Starter à 29€/mois, plan Pro à 89€/mois)
- MRR : 68 000€ — croissance 12% MoM
- NRR : 108%
- NPS : 42
- Équipe : 14 personnes (3 ingénieurs, 2 designers, 2 PMs, 7 autres)

**Stack technique** : React 18 + TypeScript + Node.js + PostgreSQL + Vercel

**Problème principal** : Fort churn à M3 (22%) — les utilisateurs créent leur premier visuel mais n'adoptent pas les fonctionnalités avancées.

## Comment utiliser les missions

### Avant de démarrer
1. Choisir sa mission selon son rôle et son niveau
2. Lire le brief complet
3. Préparer son workspace Claude Code (CLAUDE.md + outils)

### Pendant la mission
- Chaque mission a des **étapes** avec un objectif clair
- Des **ressources** pointent vers les prompts et skills pertinents
- Des **critères de réussite** permettent d'auto-évaluer

### Après la mission
- Comparer ses livrables aux **exemples de référence** dans `/examples/`
- Remplir la grille d'auto-évaluation dans `study-plan/self-evaluation.md`
- Identifier ce qui peut être réutilisé dans son workflow réel
