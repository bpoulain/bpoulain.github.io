Implémente une feature de façon complète, testée et documentée.

## Contexte

Avant de commencer :
1. Lis le CLAUDE.md pour comprendre la stack et les conventions
2. Lis les fichiers existants dans les répertoires concernés
3. Identifie les patterns à suivre

## Phase 1 — Architecture (STOP avant de coder)

Analyse l'existant et propose :
- L'approche d'implémentation
- Les fichiers à créer/modifier
- Les dépendances identifiées
- Les risques techniques potentiels
- L'estimation de complexité

**→ Attends ma validation avant de continuer.**

## Phase 2 — Implémentation

Sur la base de l'architecture validée :
1. Backend : API, logique métier, gestion des erreurs
2. Frontend : composants, états, UX
3. Gestion des cas d'erreur et edge cases

## Phase 3 — Tests

- Tests unitaires pour les fonctions critiques
- Tests d'intégration pour les endpoints
- Tests de composants (si frontend)

## Phase 4 — Documentation

- Mise à jour du README si API publique modifiée
- JSDoc sur les fonctions/composants publics
- Note dans `memory/decisions.md` si décision technique significative

## Phase 5 — Commit

Commit avec message structuré :
`type(scope): description courte`
Types : feat / fix / refactor / test / docs / chore
